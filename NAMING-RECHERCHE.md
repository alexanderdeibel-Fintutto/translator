# Naming-Recherche: Dachmarke fuer Fintutto Translator

**Datum:** 2026-03-16
**Status:** Entwurf / Diskussionsgrundlage

## Ausgangslage

Der Fintutto Translator umfasst mittlerweile 4 Produktlinien mit 10 App-Varianten:

### Produktlinie 1: Guide Translator (Tourismus)
- Stadtfuehrer, Reiseagenturen, Kreuzfahrt, Events/Museen/Konferenzen
- Eigene Marke fuer den Tourismusbereich

### Produktlinie 2: School Translator (Bildung)
- Lehrer-App + Schueler-App
- Echtzeit-Uebersetzung im Klassenzimmer

### Produktlinie 3: Authority/Amt Translator (Behoerden)
- Sachbearbeiter-App + Besucher-App
- Kommunikation am Behoerden-Schalter

### Produktlinie 4: NGO/Refugee Translator (Soziales)
- Helfer-App + Klient-App
- Unterstuetzung bei Beratung und Betreuung

**Problem:** "Fintutto Translator" ist als Dachmarke zu klein/limitierend fuer diesen Umfang.

## Empfehlung

### Top-Empfehlung: **fintutto.world**

**Gruende:**
1. Bestehende Marke "Fintutto" wird beibehalten (Brand-Konsistenz)
2. TLD `.world` drueckt den globalen Anspruch aus
3. Kein Namenskonflikt mit bestehenden Firmen/Marken
4. Flexible Subdomain-Struktur moeglich:
   - `guide.fintutto.world` - Guide Translator
   - `school.fintutto.world` - School Translator
   - `amt.fintutto.world` - Authority Translator
   - `ngo.fintutto.world` - NGO Translator
   - `app.fintutto.world` - Consumer App
5. Passt zum bestehenden `fintutto.cloud` Setup
6. `.world` Domains sind guenstig (ca. 2-5 EUR/Jahr)

### Alternative: **VoiceWorld by Fintutto**

Falls eine eigenstaendige Produktmarke gewuenscht ist:
- Beschreibt die Kernfunktion (Stimme + Welt)
- Kein bekannter Markenkonflikt
- Domain-Verfuegbarkeit muss noch geprueft werden

### Vercel CLI Setup (pro App)

```bash
# 1. In App-Verzeichnis wechseln und Vercel-Projekt verknuepfen
cd apps/school-teacher
vercel link --project tl-school-teacher

# 2. Custom Domain hinzufuegen
vercel domains add school-teacher.fintutto.world

# 3. Environment Variables setzen (Vercel Dashboard oder CLI)
vercel env add VITE_SUPABASE_URL production
vercel env add VITE_SUPABASE_ANON_KEY production
vercel env add VITE_GOOGLE_CLOUD_API_KEY production
vercel env add VITE_AZURE_TRANSLATE_KEY production
vercel env add VITE_AZURE_TRANSLATE_REGION production
vercel env add VITE_STRIPE_PUBLISHABLE_KEY production
vercel env add VITE_GA_MEASUREMENT_ID production
vercel env add VITE_SENTRY_DSN production
vercel env add VITE_ADMIN_API_URL production
vercel env add VITE_ADMIN_API_KEY production

# Wiederholen fuer alle 12 Apps
```

### Naming-Konventionen (Kurzreferenz)

| Kontext | Muster | Beispiel |
|---------|--------|----------|
| URL / Subdomain | `{rolle}.fintutto.world` | `medical-staff.fintutto.world` |
| Vercel-Projekt | `tl-{variant}` | `tl-medical-staff` |
| App-ID (Capacitor) | `cloud.fintutto.{market}.{rolle}` | `cloud.fintutto.medical.staff` |
| iOS URL Scheme | `{market}translator{rolle}` | `medicaltranslatorstaff` |
| Icon-Verzeichnis | `public/icons/{market}/` | `public/icons/medical/` |
| Build-Script | `build:{variant}` | `build:medical-staff` |
| Dev-Port | Siehe app.config.ts | 5190-5201 |
