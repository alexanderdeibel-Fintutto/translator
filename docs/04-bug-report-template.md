### Severity-Definitionen

| Level | Beschreibung | Beispiel |
|---|---|---|
| **Kritisch** | App stuerzt ab, Datenverlust, Sicherheitsluecke | Payment wird doppelt ausgefuehrt |
| **Hoch** | Kernfunktion blockiert, kein Workaround | Login funktioniert nicht |
| **Mittel** | Feature eingeschraenkt, Workaround moeglich | Filter zeigt falsche Ergebnisse |
| **Niedrig** | Kosmetisch, Tippfehler, minimale UX-Probleme | Button-Text abgeschnitten |
