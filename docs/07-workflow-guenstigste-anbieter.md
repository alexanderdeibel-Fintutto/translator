# Workflow: QA-Tester ueber die zwei guenstigsten Plattformen

---

## Option A: Jobbers (0% Kommission)

**Warum Jobbers:** Keine Gebuehren fuer Client UND Freelancer. Ihr zahlt nur den vereinbarten Stundensatz -- nichts geht an die Plattform.

### Schritt-fuer-Schritt Workflow

```
Woche 1: Vorbereitung + Hiring
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Tag 1-2: Vorbereitung
  ├── 1. Account auf jobbers.io erstellen (kostenlos)
  ├── 2. NDA-Vorlage finalisieren (docs/06-nda-vorlage-qa-tester.md)
  ├── 3. 30 Testprotokolle in Google Sheets uebertragen
  ├── 4. Bug-Report-Template bereitstellen (docs/04-bug-report-template.md)
  └── 5. Test-Accounts und Zugangsdaten vorbereiten

Tag 2-3: Job posten
  ├── 1. Job-Posting erstellen (Vorlage: docs/05-upwork-job-posting.md anpassen)
  ├── 2. Budget: $12-20/Std. (da keine Plattform-Gebuehren)
  ├── 3. Kategorie: QA Testing / Software Testing
  └── 4. Freelancer-Profile pruefen und Shortlist erstellen

Tag 4-5: Auswahl + Onboarding
  ├── 1. Top 2-3 Kandidaten interviewen (kurzes Video-Call)
  ├── 2. Kleinen bezahlten Probetask geben (1-2 Stunden, 2-3 Testprotokolle)
  ├── 3. Bug-Report-Qualitaet bewerten
  ├── 4. NDA unterzeichnen lassen (digital via DocuSign/HelloSign)
  └── 5. Zugangsdaten teilen (nach NDA-Unterzeichnung)

Woche 2-3: Testing-Durchfuehrung
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ├── 1. Tester fuehrt Testprotokolle 1-30 durch
  ├── 2. Bugs werden ueber Bug-Report-Template dokumentiert
  ├── 3. Taeglicher/zweitaeglicher kurzer Check-in (Slack/Chat)
  ├── 4. Ihr fixt Bugs laufend
  └── 5. Tester verifiziert Bugfixes (Re-Testing)

Woche 3: Abschluss
━━━━━━━━━━━━━━━━━━
  ├── 1. Tester erstellt Abschlussbericht
  ├── 2. Zahlung ueber Jobbers-Plattform
  ├── 3. Tester loescht Zugangsdaten (gemaess NDA)
  └── 4. Optional: Bewertung auf Jobbers hinterlassen
```

### Kostenbeispiel Jobbers

| Posten | Berechnung | Kosten |
|---|---|---|
| Probetask | 2 Std. x $15 | $30 |
| Testing (30 Protokolle) | 40 Std. x $15 | $600 |
| Re-Testing (Bugfixes) | 10 Std. x $15 | $150 |
| Plattform-Gebuehr | 0% | **$0** |
| **Gesamt** | | **$780** |

## Option B: Upwork ($12-15/Std. mit 3-5% Client-Fee)

**Warum Upwork:** Groesste Auswahl an QA-Testern weltweit, Bewertungssystem, Escrow-Zahlungsschutz, schnelles Matching.

### Schritt-fuer-Schritt Workflow

```
Woche 1: Vorbereitung + Hiring
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Tag 1: Vorbereitung
  ├── 1. Upwork-Client-Account erstellen und verifizieren
  ├── 2. Zahlungsmethode hinterlegen (Kreditkarte oder PayPal)
  ├── 3. NDA, Bug-Template und Testprotokolle vorbereiten
  └── 4. Job-Posting erstellen (docs/05-upwork-job-posting.md)

Tag 1-2: Job posten
  ├── 1. Job als "Intermediate" einstellen
  ├── 2. Budget: $15-25/Std. ODER Fixed Price $750-1.500
  ├── 3. Screening-Fragen aktivieren (siehe Posting-Vorlage)
  ├── 4. "Invite to Apply" nutzen: Freelancer mit guten
  │     QA-Profilen direkt einladen
  └── 5. Upwork schlaegt automatisch passende Freelancer vor

Tag 2-4: Auswahl
  ├── 1. Bewerbungen sichten (typisch 10-30 innerhalb 24h)
  ├── 2. Filtern nach:
  │     ├── Job Success Score > 90%
  │     ├── Mindestens 100+ Stunden auf Upwork
  │     ├── Relevante QA-Erfahrung
  │     └── Verfuegbare Geraete (iOS + Android)
  ├── 3. Top 3 Kandidaten kurz interviewen (Upwork-Chat oder Video)
  └── 4. Bezahlten Probetask vergeben (1-2 Stunden)

Tag 4-5: Onboarding
  ├── 1. Contract auf Upwork erstellen (Hourly oder Fixed)
  ├── 2. NDA ueber Upwork teilen und unterzeichnen lassen
  │     (Upwork hat eingebaute NDA-Funktion unter "Offer Terms")
  ├── 3. Testprotokolle und Bug-Template teilen
  ├── 4. Zugangsdaten ueber sicheren Kanal teilen
  └── 5. Kommunikationskanal festlegen (Upwork Messages reicht)

Woche 2-3: Testing-Durchfuehrung
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ├── 1. Bei Hourly-Contract: Upwork Time Tracker aktivieren
  │     (macht automatisch Screenshots als Arbeitsnachweis)
  ├── 2. Tester arbeitet Testprotokolle 1-30 ab
  ├── 3. Bugs werden ueber Bug-Report-Template dokumentiert
  ├── 4. Woechentliches Upwork-Billing (automatisch)
  └── 5. Re-Testing nach Bugfixes

Woche 3: Abschluss
━━━━━━━━━━━━━━━━━━
  ├── 1. Abschlussbericht vom Tester
  ├── 2. Contract auf Upwork schliessen
  ├── 3. Bewertung hinterlassen (wichtig fuer den Freelancer!)
  ├── 4. Tester loescht Zugangsdaten
  └── 5. Optional: Freelancer fuer kuenftige Test-Zyklen "favorisieren"
```

### Kostenbeispiel Upwork

| Posten | Berechnung | Kosten |
|---|---|---|
| Probetask | 2 Std. x $15 | $30 |
| Testing (30 Protokolle) | 40 Std. x $15 | $600 |
| Re-Testing (Bugfixes) | 10 Std. x $15 | $150 |
| Upwork Client-Fee (3-5%) | 5% von $780 | **$39** |
| **Gesamt** | | **$819** |

## Vergleich: Jobbers vs. Upwork

| Kriterium | Jobbers | Upwork |
|---|---|---|
| **Gesamtkosten** | ~$780 (0% Fee) | ~$819 (3-5% Fee) |
| **Freelancer-Auswahl** | Kleiner Pool | Riesiger Pool, 12 Mio.+ |
| **Bewertungssystem** | Basis | Detailliert (Job Success Score) |
| **Zahlungsschutz** | Vorhanden | Escrow + Time Tracker |
| **Eingebautes NDA** | Nein | Ja (in Offer Terms) |
| **Geschwindigkeit** | Mittel | Schnell (24h erste Bewerbungen) |
| **Risiko** | Etwas hoeher (kleinere Community) | Niedriger (etablierte Plattform) |

### Empfehlung

- **Jobbers** wenn ihr maximale Kostenersparnis wollt und bereit seid, etwas mehr Zeit in die Kandidatensuche zu investieren.
- **Upwork** wenn ihr schnell starten wollt, Wert auf Bewertungen/Zahlungsschutz legt und die 3-5% Aufpreis akzeptiert.
- **Hybrid-Ansatz:** Auf beiden Plattformen parallel posten und den besten Kandidaten nehmen, unabhaengig von der Plattform.
