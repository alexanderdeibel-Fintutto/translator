## 3. FULL DESCRIPTION (Hunter/Maker Comment)

### Maker Comment (fuer den Launch-Post)

```
Hey Product Hunt! 👋

We built GuideTranslator because we saw tour guides struggling with
multilingual groups, teachers unable to communicate with refugee
students, and conference organizers paying $10,000+/day for
simultaneous interpreters covering just 3 languages.

**The problem:** Real-time group translation doesn't exist --
or it costs a fortune and needs perfect internet.

**Our solution:** GuideTranslator lets one speaker broadcast live
translations to unlimited listeners in 45 languages. Each listener
picks their language and reads + hears the translation on their
own phone. No app install -- just scan a QR code.

**The killer feature:** It works WITHOUT internet. 🤯

We built a 4-tier transport system:
- ☁️ Cloud mode (Supabase Realtime -- best quality)
- 📡 Hotspot mode (speaker creates own WiFi network)
- 📶 Bluetooth LE mode (zero infrastructure needed)
- 📴 Full offline mode (on-device ML translation)

Whether you're a tour guide in ancient ruins with no cell signal,
a teacher in a school with bad WiFi, or an NGO worker in a refugee
camp -- it just works.

**What makes it different from Google Translate?**
- Google: 1-to-1. GuideTranslator: 1-to-500.
- Google: needs internet. We work offline.
- Google: no broadcast. We do real-time broadcast via QR code.
- Google: no voice output. We have Neural2 & Chirp 3 HD voices.

**Built with:**
React 18 + TypeScript, Capacitor (iOS/Android), BLE GATT,
Opus-MT (on-device translation), Web Speech API, Google Cloud
TTS with Neural2/Chirp 3 HD voices.

**Free tier available** -- try it right now at translator.fintutto.cloud

We'd love your feedback! 🙏
```

## 10. ANTWORT-VORLAGEN FUER PRODUCT HUNT KOMMENTARE

### "How is this different from Google Translate?"

```
Great question! The main differences:

1. **1-to-many**: Google is 1-to-1. We broadcast to hundreds
   of listeners simultaneously.
2. **No internet needed**: We work via Bluetooth LE or local
   Hotspot. Google needs internet for everything.
3. **QR Code join**: Listeners don't install anything. They
   scan a code and pick their language.
4. **HD voice output**: Google Neural2 & Chirp 3 HD voices,
   not robotic TTS.
5. **Session export**: Full transcript of every session.

Think of it as "Google Translate meets Zoom meets offline" 😊
```

### "Does it really work offline?"

```
Yes! We built a 4-tier transport system:

1. **Cloud** (best quality) -- uses Supabase Realtime
2. **Hotspot** -- speaker creates own WiFi, no router needed
3. **Bluetooth LE** -- direct device-to-device, ~30m range
4. **Full offline** -- on-device ML models (Opus-MT via
   Transformers.js)

The app automatically detects connectivity and falls back
through the tiers. In full offline mode, translation quality
is slightly lower (ML models vs. cloud API), but it absolutely
works -- we've tested it in basements, forests, and boats.
```

### "What about privacy / GDPR?"

```
Privacy is core to our design:

- **No account required** for listeners
- **No conversation data stored** on our servers
- **AES-256-GCM encryption** for all live sessions
- **Offline modes** = data never leaves the device
- **GDPR compliant** -- we're a German company (EU-based)
- **Session transcripts** are only stored locally unless
  the speaker explicitly exports them

In BLE/Hotspot mode, there's literally no server involved.
Data flows directly between devices.
```

### "What languages do you support?"

```
Currently 45 languages for live translation, including:

🇪🇺 European: DE, EN, FR, ES, IT, PT, NL, PL, CZ, RO, HU,
   EL, SV, DA, BG, HR, SR, SK, SL, NO, FI
🌍 Middle East: AR, FA (Farsi/Dari), HE, TR, KU (Kurdish),
   PS (Pashto)
🌏 Asian: ZH, JA, KO, HI, TH, VI, ID, MS, FIL, BN, UR
🌍 African: TI (Tigrinya), AM (Amharic), SO (Somali), SW
🇺🇦 Plus: UK (Ukrainian), RU, SQ (Albanian)

Full RTL support for Arabic, Farsi, Urdu, and Hebrew.
Offline mode supports 100+ language pairs via Opus-MT models.
```
