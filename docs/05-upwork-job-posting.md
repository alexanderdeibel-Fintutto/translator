## Posting-Vorlage (Englisch - fuer internationale Tester)

---

### Titel

**Manual QA Tester Needed for Multilingual Translation App (Mobile + Web)**

---

### Beschreibung

We are looking for an experienced **manual QA tester** to test our translation/finance app **Fintutto Translator** across web and mobile platforms (iOS & Android via Capacitor).

**About the App:**
- Multilingual translation app with financial document focus
- Built with React + Vite (Web), Capacitor (iOS/Android)
- Supabase backend with real-time features
- Admin dashboard included

**What You Will Do:**
- Execute **30 existing test protocols** (provided as structured test cases)
- Test critical user flows: registration, login, document translation, payment
- Test across multiple devices and browsers
- Document all bugs using our bug report template (provided)
- Verify bug fixes after developer resolution
- Provide a summary report after each test cycle

**Requirements:**
- 2+ years of manual QA testing experience
- Experience testing mobile apps (iOS & Android)
- Experience testing web applications
- Familiarity with bug tracking tools (GitHub Issues, Jira, or similar)
- Strong attention to detail
- Good written English (bug reports must be clear and reproducible)
- Ability to sign an NDA before starting

**Nice to Have:**
- Experience with translation/localization apps
- German language skills
- Experience with Capacitor/hybrid apps
- ISTQB certification

**Project Details:**
- **Duration:** ~30-60 hours (initial test cycle)
- **Timeline:** 2-3 weeks
- **Budget:** $15-25/hour (depending on experience and location)
- **Type:** Fixed-price or hourly (negotiable)

**To Apply, Please Include:**
1. Brief description of your QA testing experience
2. Example of a bug report you have written
3. Devices/browsers you have available for testing
4. Your hourly rate
5. Availability (hours per week)

We will provide all test protocols, access credentials, and a bug report template.
